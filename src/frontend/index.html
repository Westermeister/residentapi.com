Documentation

Authorization Header

Listen here, stranger! During your registration, you're required to specify both a username and a password. As with any website, the former identifies you, while the latter authenticates you!

Callin' this here API involves a simple scheme called Basic Authentication. All you gotta do is take your username and password, then smush them together with a colon in the middle e.g. username:password. Then, encode that in Base64 and include the result as an Authorization header: 

Authorization: Basic your_base64_here

Easy as pie, ain't it, stranger?

Available Endpoints

All requests should be formatted like so:

GET https://residentapi.com/quotes[?character=optional-code][?source=optional-code]
  Authorization: Basic your_base64_here

As you can see stranger, you can optionally include a character code to get a quote from a specific person, and/or a source code to get a quote from specific source.

API Response

After making a request, you'll receive a 200 status code and JSON with 4 keys: quote, author, context, and source. For example: 

{
  "quote": "Game over.",
  "author": "Chris Redfield",
  "context": "after defeating Lucas Baker",
  "source": "Resident Evil 7"
}

If the request failed in any way, you'll get a 4xx or 5xx status code and a JSON response with an error message, like so: 

{
  "error": "Example error message"
}

Always check that the status code is 200 before proceeding your with program logic!

Limitations

The only limitation to be aware of, stranger, is that after making an API call and receiving a response, you must wait at least one second before making another call.

If you don't, you'll be slapped with a nasty 429 status code. Respect the rules of the road, stranger!
